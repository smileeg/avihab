# Habit Tracker

## Огляд

Повноцінний мобільний застосунок для трекінгу звичок, створений на React, TypeScript та Vite. Застосунок дозволяє користувачам створювати та керувати щоденними звичками, відстежувати серії виконання, заробляти XP та рівні, а також відкривати бейджі досягнень. Має mobile-first дизайн з каваї-естетикою, пастельними кольорами та emoji-іконками.

## Користувацькі вподобання

Мова спілкування: Українська

## Системна архітектура

### Архітектура фронтенду

**Фреймворк та інструменти збірки**
- React 19.2 з TypeScript для типобезпечної розробки компонентів
- Vite 7.2 для швидкого dev-сервера та оптимізованих production-збірок
- JSX трансформація з використанням сучасного `react-jsx` runtime

**Управління станом**
- Context API для глобального управління станом
- Єдиний `AppContext` надає централізований стан та дії
- Персистентність через localStorage для даних звичок
- useMemo для обчислюваних полів (stats, badges)
- Компонентний стан для форм та UI-взаємодій

**Структура компонентів**
- Архітектура на основі екранів з 5 головними екранами:
  - `HomeScreen`: Головна панель зі звичками та щоденним прогресом
  - `AddEditHabitScreen`: Форма для створення/редагування звичок
  - `HabitDetailScreen`: Індивідуальна статистика та історія звички
  - `StatsScreen`: Загальна статистика та візуалізація прогресу
  - `RewardsScreen`: Досягнення бейджів та розблокування
- Клієнтська навігація через state-based screen switching (без бібліотеки роутера)

**UI дизайн**
- Mobile-first адаптивний дизайн (max-width: 480px)
- Каваї-естетика з пастельною кольоровою палітрою та emoji-іконками
- Градієнтні фони та заокруглені кути по всьому застосунку
- Без зовнішніх UI-бібліотек - власна CSS стилізація

**Потік даних**
- Однонаправлений потік даних через Context API
- Дії модифікують стан, викликаючи ре-рендери
- Похідні дані (stats, прогрес) обчислюються через memoization
- Синхронізація localStorage при зміні даних

### Ключова логіка застосунку

**Система гейміфікації**
- Система XP: 10 XP за кожне виконання звички
- Левелінг: 100 XP за рівень
- Відстеження серій: консекутивні дні з виконаними звичками
- Система бейджів: 8 бейджів досягнень з умовами розблокування

**Відстеження прогресу**
- Масиви дат виконання для кожної звички
- Обчислення щоденного, тижневого та місячного прогресу
- Розрахунок відсотка виконання за періоди
- Візуалізація heatmap для 7-денної історії

**Управління звичками**
- Ідентифікація звичок на основі UUID (timestamp)
- Налаштовувані іконки (16 попередньо визначених emoji)
- Налаштовувані кольори (10 пастельних кольорів)
- ISO формат дат для відстеження виконання (YYYY-MM-DD)

### Моделі даних

**Тип Habit**
```typescript
{
  id: string (timestamp)
  name: string
  icon: string (emoji)
  color: string (hex color)
  createdAt: string (ISO date)
  completions: string[] (ISO dates)
}
```

**Тип Badge**
```typescript
{
  id: string
  name: string
  description: string
  icon: string (emoji)
  unlocked: boolean
  unlockedAt?: string (ISO date)
}
```

**User Stats**
- Динамічно обчислюється з виконання звичок
- Total XP, level, current streak, longest streak, total completions
- Без окремого зберігання - похідні від даних звичок

## Зовнішні залежності

### Основні залежності
- **React 19.2**: UI фреймворк з найновішими функціями
- **React DOM 19.2**: DOM рендеринг

### Інструменти розробки
- **TypeScript 5.9**: Статична перевірка типів
- **Vite 7.2**: Інструмент збірки та dev-сервер
- **ESLint 9.39**: Лінтинг коду з підтримкою TypeScript
- **@vitejs/plugin-react 5.1**: Інтеграція React для Vite

### Зберігання даних
- Browser LocalStorage для персистентності
- Storage key: `'habit-tracker-habits'`
- Без backend або бази даних - повністю клієнтський застосунок

### Конфігурація сервера
- Vite dev-сервер налаштований для:
  - Host: `0.0.0.0` (доступний по мережі)
  - Port: 5000
  - Allowed hosts: true (для Replit середовища)

### Сторонні сервіси
Немає - повністю автономний frontend-застосунок без зовнішніх API-інтеграцій.

## Важливі технічні деталі

### Логіка розрахунку серій (Streak)
- **currentStreak**: Активується тільки якщо останнє виконання сьогодні або вчора, потім рахує назад консекутивні дні
- **longestStreak**: Сканує всі унікальні дати виконання та знаходить максимальну консекутивну послідовність

### Бейджі та умови розблокування
- **Перший крок**: Виконайте першу звичку
- **Тижневий воїн**: Підтримуйте серію 7 днів
- **Майстер місяця**: Підтримуйте серію 30 днів
- **Колекціонер звичок**: Створіть 5 звичок
- **Рівень 5**: Досягніть 5 рівня
- **Рівень 10**: Досягніть 10 рівня
- **Клуб сотні**: Виконайте 100 звичок
- **Король постійності**: Виконуйте всі звички протягом тижня

## Запуск та розробка

```bash
# Встановити залежності
npm install

# Запустити dev-сервер
npm run dev

# Збірка для production
npm run build

# Попередній перегляд production збірки
npm run preview
```

Застосунок буде доступний за адресою http://localhost:5000/
